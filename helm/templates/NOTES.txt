Thank you for installing {{ .Chart.Name }}.

Your release is named {{ .Release.Name }}.

To check the status of your deployment, run:
  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

To check if your secrets have been successfully synchronized from Vault, run:
  kubectl get externalsecret {{ .Release.Name }}-app-secrets -n {{ .Release.Namespace }}

To access your application:
{{- if contains "LoadBalancer" .Values.app.service.type }}
  NOTE: It may take a few minutes for the LoadBalancer IP to be available.
  You can watch the status by running:
    kubectl get svc {{ .Release.Name }}-app-service -n {{ .Release.Namespace }} -w

  Once the EXTERNAL-IP is available, you can access the app at:
    http://<EXTERNAL-IP>:{{ .Values.app.port }}
{{- else if contains "NodePort" .Values.app.service.type }}
  Get the application URL by running these commands:
    export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ .Release.Name }}-app-service)
    export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
    echo http://$NODE_IP:$NODE_PORT
{{- else if contains "ClusterIP" .Values.app.service.type }}
  Get the application URL by running these commands:
    export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/name={{ include "school-app.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}")
    echo "Visit http://127.0.0.1:5000 to use your application"
    kubectl --namespace {{ .Release.Namespace }} port-forward $POD_NAME 5000:{{ .Values.app.port }}
{{- else }}
  Access the app via the LoadBalancer service:
    kubectl get svc {{ .Release.Name }}-app-service -n {{ .Release.Namespace }}
{{- end }}

IMPORTANT: Ensure that Vault is correctly configured and the secrets path exists in Vault for the ExternalSecret to synchronize.
